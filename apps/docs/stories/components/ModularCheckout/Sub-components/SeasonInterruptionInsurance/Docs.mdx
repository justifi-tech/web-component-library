# Authorization

---

<p>
  Authorization is performed by passing a web component token as{' '}
  <code>auth-token</code>.
</p>
<ul>
  <li>
    <strong>Web Component Token</strong>: These tokens are generated by your
    backend services using the [Web Component Tokens
    API](https://docs.justifi.tech/api-spec#tag/Web-Component-Tokens/operation/CreateWebComponentToken).
    Each token can be scoped to perform a set number of actions and is active
    for 60 minutes. When creating a web component token for this specific
    component you'll need to use the role: `read:account:account_id` or
    `write:account:account_id`. Make sure the value for `account_id` matches the
    prop you also pass separately.
  </li>
</ul>

# CSS Parts for Styling

---

The Season Interruption Insurance component exposes several CSS parts for custom styling:

## Individual Element Parts

- `text` - Text elements (description, legal disclaimer)
- `text-danger` - Error text styling
- `heading-2` - Header text styling (via Header2 component)
- `label` - Radio button labels

## Component Structure Parts

- `insurance-container` - The main container wrapping the entire component
- `insurance-loading` - The loading state container

## Content Section Parts

- `insurance-header-section` - The header/title section
- `insurance-description-section` - The product description section
- `insurance-form-section` - The form section containing radio buttons
- `insurance-legal-disclaimer-section` - The legal disclaimer section

## Form Parts

- `insurance-form` - The form element
- `insurance-radio-group` - The container for radio button options
- `input-radio` - Radio button inputs (various states available)

# Behavior when disabled

If the parent checkout's `payment_settings.insurance_payments` is `false`, this component will render nothing and emit a console warning to help with debugging. This allows you to include the component unconditionally without needing to hide it when the feature is disabled for a given checkout.
