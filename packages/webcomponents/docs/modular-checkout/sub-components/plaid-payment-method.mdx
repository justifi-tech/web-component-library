## Overview

`<justifi-plaid-payment-method>` launches Plaid Link from inside Modular Checkout so customers can connect bank accounts instantly. The component handles link token generation, success/error events, and shares the resulting bank token with the parent checkout.

### Events

- `plaid-success`: Provides the Plaid public token and metadata when customers finish linking.
- `plaid-exit`: Emits when Link closes without success and includes exit metadata.
- `error`: Surfaces integration failures (expired link token, network issues).

### Public methods

1. `open()` – Programmatically open Plaid Link (if you render a custom CTA).
2. `close()` – Force-close the Link flow.

## Theming & Layout

- Styling is limited to the wrapper since Plaid controls the Link modal; use CSS props like `--justifi-rail-spacing` to maintain consistent padding between rails.
- Provide fallback instructions for devices where Plaid is unavailable.

## Accessibility

- Plaid Link provides its own WCAG-compliant UI; this component ensures the host button has `aria-label` text describing the rail.
- Screen-reader messaging indicates when the Plaid modal opens and closes, keeping focus predictable.
