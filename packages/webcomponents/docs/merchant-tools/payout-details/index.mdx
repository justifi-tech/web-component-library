## Overview

`<justifi-payout-details>` complements `<justifi-payouts-list>` by showing metadata, funding status, line items, and links to underlying payments. Use it in dashboards or CRM workflows where operators need context before contacting merchants.

## Usage

<UsageSnippet
  title="Inspect payout"
  code={`<justifi-payout-details
  account-id="acc_123"
  auth-token="wct_reporting"
  payout-id="po_456"
/>`}
  language="html"
/>

### Events

- `error-event`: Emits when payout data cannot be loaded.

### Public methods

1. `refresh()` – Reload payout data after remediation steps.

## Theming & Layout

- Customize surfaces with `--justifi-card-surface` and `--justifi-card-border`.
- Provide roughly 480–720px of width to keep long account numbers and addresses readable.

## Accessibility

- The ledger uses semantic `<table>` markup with summary captions explaining the payout.
- Bank details include visually hidden text describing the last four digits for screen readers.
