## Overview

`<justifi-checkouts-list>` exposes a sortable table of checkout attempts so operations teams can research payment intent history. Pair it with `<justifi-checkouts-list-filters>` to let users slice data by status, amount, or created date.

## Usage

<UsageSnippet
  title="Basic list with filters"
  code={`<justifi-checkouts-list-filters></justifi-checkouts-list-filters>
<justifi-checkouts-list
  account-id="acc_123"
  auth-token="wct_reportingToken"
/>`}
  language="html"
/>

### Custom columns

Pass a comma-separated list to the `columns` prop (`created_at,payment_amount,status`) to match the data points your operators expect.

### Events

- `click-event`: Emitted when a row or control is clicked. `event.detail.name` indicates the source.
- `error-event`: Fires when the list cannot load data due to network/auth issues.

### Methods

1. `refresh()` – Re-fetches data. Combine with filter changes for dynamic searches.
2. `exportCSV()` – (Coming soon) Provide your own download button and call this method for exports.

## Theming & Layout

- Adjust table chrome with CSS custom properties like `--justifi-table-header-bg`, `--justifi-table-border-color`, and status-specific colors (`--justifi-status-pill-success`, `--justifi-status-pill-warning`).
- Wrap the component in a responsive container; below 960px the table enables horizontal scrolling, so keep surrounding padding generous.

## Accessibility

- Table headers use `<th scope="col">` and rows use `<tr role="row">` for assistive tech.
- Status badges include `aria-label` text describing the underlying state (e.g., “Completed checkout”).
- Keyboard users can navigate rows via arrow keys; focus indicators adhere to the default outline tokens.
