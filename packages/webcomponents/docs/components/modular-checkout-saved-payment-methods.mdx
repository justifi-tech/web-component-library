## Overview

`<justifi-saved-payment-methods>` lets returning customers pick an existing tokenized method before submitting a checkout. It integrates with Modular Checkout to sync selection state and gracefully falls back to new method entry when none exist.

## Usage

<UsageSnippet
  title="Place saved methods above the card form"
  code={`<justifi-modular-checkout
  account-id="acc_123"
  auth-token="wct_checkout"
  checkout-id="chk_456"

  <justifi-saved-payment-methods slot="payment-methods" />
  <justifi-modular-checkout-card-form slot="payment-methods" />
  <justifi-modular-checkout-summary slot="summary" />

### Events

- `selection-changed`: Emits when customers choose a different saved method.
- `delete-requested`: Fires when someone clicks delete on a saved method (confirm server-side permissions).

## Theming & Layout

- Customize chips/cards with CSS props: `--justifi-payment-chip-border`, `--justifi-payment-chip-active-border`, and `--justifi-payment-chip-padding`.
- Provide at least 320px of width; on mobile the chips stack vertically.

## Accessibility

- Chips behave like radio buttons with `role="radiogroup"` and `aria-checked` states.
- Delete actions confirm via `aria-live` messaging once a method is removed.
