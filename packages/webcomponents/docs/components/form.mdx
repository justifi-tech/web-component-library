## Overview

`<justifi-form>` is a low-level building block that wraps inputs, handles validation summaries, and emits standardized submit events. It powers many domain-specific components but can also be embedded inside bespoke flows when you want JustiFi’s UX without the domain logic.

## Usage

<UsageSnippet
  title="Custom form content"
  code={`<justifi-form
  heading="Verify business contact"
  description="We’ll use this info to keep payout notifications flowing."
  submit-label="Save contact"
>
  <label slot="fields">
    Contact name
    <input type="text" name="contact_name" required />
  </label>
</justifi-form>`}

/>

### Events

- `submitted`: Fires when the form submits (bubbles detail with form data payload).
- `error-event`: Emitted if client-side validation fails.

### Public methods

1. `submit()` – Trigger validation + submission.
2. `reset()` – Clear field values and validation messages.

## Theming & Layout

- Use CSS vars such as `--justifi-form-surface`, `--justifi-form-border-color`, and `--justifi-form-spacing` to align with your shell.
- The component stretches to 100% width but is typically constrained to 320–520px for readability.

## Accessibility

- Focus automatically moves to the validation summary when errors occur.
- Provide labelled inputs inside the `fields` slot; the component will associate summary items with field names.
