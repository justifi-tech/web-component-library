## Overview

`<justifi-business-form>` orchestrates multi-step onboarding for business representatives, beneficial owners, and supporting documents. Embed it when you want a single component to lead users through KYC requirements without maintaining bespoke forms.

### Events

- `submitted`: fires when the flow completes; payload includes representative + ownership data.
- `step-changed`: emits whenever the form advances to the next required section.
- `error-event`: emits validation or network issues for logging surfaces.

### Public methods

1. `next()` – Move to the next step programmatically (only when current step is valid).
2. `previous()` – Return to the earlier step (useful for custom navigation controls).

## Theming & Layout

- Align the shell with CSS variables like `--justifi-form-surface`, `--justifi-form-border-color`, and `--justifi-form-spacing`.
- Keep the component between 360px and 640px wide; longer forms automatically paginate to minimize scrolling.

## Accessibility

- Each step uses semantic headings to describe the current requirement and keeps focus within the visible panel.
- Inline errors announce via `aria-live="assertive"` and reference the owning input with `aria-describedby`.
- Screen-reader only text clarifies why sensitive fields (SSN, ownership %) are collected.
