## Overview

`<justifi-modular-checkout>` lets you assemble a checkout experience using specialized subcomponents (card forms, bank forms, summary, saved payment methods, etc.). It handles context sharing (auth tokens, account IDs), state management, and submission orchestration.

## Usage

<UsageSnippet
  title="Modular layout with summary"
  code={`<justifi-modular-checkout

### Events

- `completed`: Fired when all nested components finish and a payment succeeds.
- `error`: Emitted when nested components report a failure (validation, network, etc.).
- `step-changed`: Surfaces stepper transitions for analytics.

### Public methods

1. `submit()` – Programmatically trigger the checkout CTA.
2. `reset()` – Reset the flow back to the first step and clear transient state.

## Theming & Layout

- Use CSS props like `--justifi-modular-surface`, `--justifi-modular-border-radius`, and `--justifi-modular-spacing` to keep the container in sync with your brand.
- Slots expose `::part(summary)` and `::part(payment-methods)` so you can adjust spacing for each region.
- The layout stacks sections vertically on mobile (&lt;640px) and aligns them side-by-side on larger breakpoints.

## Accessibility

- Handles focus management when switching between steps or nested components.
- CTA buttons expose `aria-busy` while processing.
- Summary region uses semantic lists for line items so assistive tech can read totals clearly.
