## Overview

`<justifi-payment-method-options>` exposes the same selector UI used inside Storybook examples. It pairs well with `<justifi-modular-checkout>` when you need more control over where the payment rail picker appears.

## Usage

<UsageSnippet
  title="Standalone selector"
  description="Let customers choose a rail before revealing underlying components."
  code={`<justifi-payment-method-options
  selected="card"
  options='[
    {"id":"card","label":"Credit or debit card"},
    {"id":"ach","label":"Bank account"},
    {"id":"wallet","label":"Digital wallet"}
  ]'
></justifi-payment-method-options>`}
  language="html"
/>

## Theming & Layout

- Adjust spacing/typography with CSS vars such as `--justifi-rail-spacing`, `--justifi-rail-border-radius`, and `--justifi-rail-active-border-color`.
- Provide at least 280px width; in list mode each option spans the full width, whereas cards wrap on desktop.

## Accessibility

- Options behave like radio buttons with `role="radiogroup"` and `aria-checked` states.
- Keyboard users can toggle rails with arrow keys; focus is always visible with default outlines.
