## Overview

`<justifi-payment-transactions-list>` exposes the granular events tied to each payment—captures, adjustments, refunds, and disputes. Finance teams can use it to reconcile ledger entries or export data for accounting.

## Usage

<UsageSnippet
  title="Transaction history"
  description="Filter by payment or customer when embedding the ledger."

### Events

- `filter-changed`: emits whenever the internal filter UI updates.
- `error-event`: surfaces API or token failures.

### Public methods

1. `refresh()` – Re-run the ledger query.

## Theming & Layout

- Share the same CSS variables as other reporting tables: `--justifi-table-header-bg`, `--justifi-table-border-color`, and `--justifi-table-row-hover-bg`.
- Provide at least 960px of width for the amount, status, and created-at columns; the component scrolls responsively when narrower.

- Amounts include visually hidden labels that describe currency and sign.
- Status badges expose descriptive `aria-label` text.
- Keyboard navigation matches the behavior of `<justifi-payments-list>`, with arrow-key row traversal and sticky headers.
