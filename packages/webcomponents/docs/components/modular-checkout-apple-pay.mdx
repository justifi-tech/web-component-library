## Overview

Use `<justifi-apple-pay>` when you want to offer Apple Pay directly inside Modular Checkout. The component renders Apple-branded buttons, verifies device/browser compatibility, and pipes the resulting token back to the parent checkout.

## Usage

<UsageSnippet
  title="Add Apple Pay as a rail"
  code={`<justifi-modular-checkout
  account-id="acc_123"
  auth-token="wct_checkout"
  checkout-id="chk_456"
>
  <justifi-apple-pay slot="payment-methods" />
  <justifi-modular-checkout-summary slot="summary" />
</justifi-modular-checkout>`}

### Events

- `availability-changed`: Notifies the parent when Apple Pay is disabled for the current device/browser.
- `error`: Surfaces tokenization failures (network, customer cancel, etc.).

### Public methods

1. `begin()` – Trigger the Apple Pay sheet programmatically if you hide the built-in button.

## Theming & Layout

- Styling is intentionally limited to keep Apple compliance; you can control surrounding padding with `--justifi-apple-pay-padding`.
- Provide at least 200px of width so the wallet button renders at the recommended size.

## Accessibility

- The button exposes `aria-label="Apple Pay"` and passes WCAG contrast tests per Apple’s guidelines.
- When Apple Pay isn’t available, the component announces the fallback message via `role="status"`.
