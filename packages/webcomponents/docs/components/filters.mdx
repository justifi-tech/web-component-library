## Overview

`<justifi-filters>` emits structured filter payloads that you can feed into `<justifi-payments-list>`, `<justifi-checkouts-list>`, or any other reporting table. It renders dropdowns, chips, and date pickers that mirror Storybook’s experience.

## Usage

<UsageSnippet
  title="Attach to payments list"
  code={`<justifi-filters
  account-id="acc_123"
  auth-token="wct_reporting"
  target="payments"
  presets='{"status":"succeeded"}'
></justifi-filters>`}

### Events

- `filters-changed`: Emits whenever a user updates a control; payload includes normalized filter values.

### Public methods

1. `clear()` – Reset to default state.
2. `apply(values)` – Programmatically update filters (useful for deep links).

## Theming & Layout

- Set CSS vars such as `--justifi-filters-surface`, `--justifi-filters-border`, and `--justifi-filters-spacing` to align with surrounding cards.
- At widths below 640px the component collapses filters into an accordion; ensure there’s enough padding for comfortable tap targets.

- Filter controls rely on native inputs, so keyboard navigation and focus ordering follow standard HTML expectations.
- Active filter chips expose `aria-label` text for removal actions.
