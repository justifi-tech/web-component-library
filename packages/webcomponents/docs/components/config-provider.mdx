## Overview

`<justifi-config-provider>` keeps your markup tidy by providing shared configuration (auth tokens, locales, formatting rules) to every nested component. Wrap dashboards or flows where many JustiFi elements appear together.

## Usage

<UsageSnippet
  title="Wrap multiple components"
  description="Provide one place for tokens and locale."
  code={`<justifi-config-provider
  auth-token="wct_shared"
  account-id="acc_123"
  locale="en-US"
  environment="sandbox"

  <justifi-checkouts-list />
  <justifi-payments-list />
</justifi-config-provider>`}

### Events

No direct events; downstream components emit their own events with inherited context.

### Public methods

- None. Configuration changes happen via attribute updates (React/Stencil bindings will cause nested components to re-render with the new context).

## Accessibility

- Because the provider has no UI, it does not affect focus order. Children inherit locale and formatting, ensuring consistent messaging for assistive tech.
