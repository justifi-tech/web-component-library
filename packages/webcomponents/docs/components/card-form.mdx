## Overview

`<justifi-card-form>` collects cardholder data using hosted fields, tokenizes the result, and emits submit events. Use it when you want a lightweight component for card capture without the rest of Modular Checkout.

## Usage

<UsageSnippet
  title="Drop-in card form"
  code={`<justifi-card-form
  account-id="acc_123"
  auth-token="wct_card"
  locale="en-US"
  show-save-payment-method
/>`}

### Events

- `tokenized`: Emits the payment method token once the form validates and tokenization succeeds.
- `error`: Surfaces validation or network errors with `code`/`message`.

### Public methods

1. `submit()` – Trigger tokenization manually.
2. `focus()` – Move focus into the hosted field experience.

## Theming & Layout

- Customize containers using CSS props: `--justifi-form-surface`, `--justifi-form-border-radius`, `--justifi-form-spacing`.
- Hosted fields remain responsive down to 320px; below that, inputs stack vertically.

## Accessibility

- Hosted fields provide semantic labels and `aria-invalid` states; the component ensures focus returns to the parent after submission.
- Screen-reader announcements describe required fields and input formats.
